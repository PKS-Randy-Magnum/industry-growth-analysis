# Pipeline & operations

Detailed reference for `run.py`, data exports, and maintenance scripts.

For **why** BEA and BLS are paired (crosswalk provenance), what date range is bundled in the repo, and high-level limitations, start with [README.md](../README.md).

## Project structure

```
industry-growth-inflation-analysis/
├── config/
│   ├── industries.yaml           # Plot filters and date windows
│   ├── bea_bls_crosswalk.csv     # BEA industry → CES 8-digit codes
│   └── bea_industry_registry.csv # BEA line_id registry
├── data/raw/                     # Legacy CSV exports (reference only)
├── data/snapshots/               # API-derived tidy CSVs + manifest.json
├── data/excel/                   # Excel-friendly panels (full + excl_trust_funds)
├── data/processed/               # SQLite DB, window-filtered CSVs per profile
├── scripts/export_snapshots.py   # Standalone snapshot export
├── sql/                          # Schema, views, analytical queries
├── src/etl/
│   ├── parse_crosswalk.py        # Optional docx → crosswalk CSV rebuild
│   ├── fetch_bls.py / fetch_bea.py
│   └── aggregate_bls.py          # Multi-series CES → BEA grain
└── run.py                        # One-command pipeline
```

## CLI options

| Flag | Default | Description |
|------|---------|-------------|
| `--bea-source` | `api` | `api` or `csv` |
| `--bls-source` | `api` | `api` or `csv` |
| `--start` | `2019-Q1` | Analysis/plot window start |
| `--end` | current quarter | Analysis/plot window end |
| `--refresh` | off | Force API re-fetch; sets api sources and full range |
| `--offline` | off | Use legacy CSVs in `data/raw/` |
| `--rebuild-db` | off | Delete and recreate SQLite from scratch |
| `--export-snapshots` / `--no-export-snapshots` | on | Write `data/snapshots/` and `data/excel/` after run |

### Examples

```bash
python run.py --refresh
python run.py --offline --start 2021-Q1 --end 2022-Q4
python run.py --rebuild-db
python run.py --start 2021-Q1 --end 2022-Q4
```

## Snapshots & Excel

After `python run.py`, see `data/snapshots/manifest.json`. BEA and BLS stay in separate files.

| BEA (`data/excel/{profile}/`) | BLS (`data/excel/{profile}/`) |
|-------------------------------|-------------------------------|
| `bea_growth_quarterly.csv` | `bls_quarterly_growth.csv` |
| `bea_growth_endpoint.csv`, `bea_growth_yearly_q1.csv` | `bls_growth_endpoint.csv`, `bls_growth_yearly_q1.csv` |
| `bea_pivot_*.csv` | `bls_pivot_*.csv` |

Also in `data/snapshots/{profile}/` (long format). UTF-8 BOM for Excel.

## Validation scripts

```bash
python scripts/check_api_keys.py      # verify .env keys before refresh
python scripts/validate_crosswalk.py  # CES series existence + BLS aggregation vs legacy CSV
python scripts/validate_bea_growth.py # BEA snapshot growth self-consistency + optional API check
```

| Output | Meaning |
|--------|---------|
| `outputs/crosswalk_validation.json` | Series coverage counts and metric-level comparison vs `data/raw/BLS Data.csv` (committed) |
| `outputs/bea_growth_validation.json` | Local BEA growth audit report (gitignored; regenerated by `validate_bea_growth.py`) |

## Regenerate crosswalk

The committed `config/bea_bls_crosswalk.csv` is the source of truth. To rebuild from a local CES mapping document:

```powershell
$env:CES_DOCX_PATH = "C:\path\to\CES CODES.docx"
python -m src.etl.parse_crosswalk
python scripts/validate_crosswalk.py
```

## CES series ID construction

[BLS CES](https://www.bls.gov/ces/) uses **8-digit industry codes** (e.g. `31336200`). Full series IDs:

```
CES + 31336200 + 01  →  employment
CES + 31336200 + 03  →  avg hourly earnings
CES + 31336200 + 11  →  avg weekly earnings
```

The crosswalk stores base codes; `expand_series_ids()` generates all three datatypes.

## SQL highlights

- `bea_bls_crosswalk` — explicit BEA ↔ CES mapping
- `v_bea_bls_comparison` — joined on `bea_line_id` via crosswalk
- `v_crosswalk_coverage` — flags industries without CES codes

## Extending data

**BEA (API, default):** Fetches chain-type quantity index (Table 8) and price index (Table 11), computes QoQ growth in Python. Set `BEA_API_KEY`, run `python run.py` or `--refresh`. Use `--rebuild-db` after pipeline upgrades.

**BEA (CSV):** Legacy `data/raw/` files are reference only.

**BLS:** Run `validate_crosswalk.py` once to warm the API cache, then `python run.py`.

SQLite at `data/processed/industry_analysis.db` accumulates history; `--rebuild-db` starts fresh.
